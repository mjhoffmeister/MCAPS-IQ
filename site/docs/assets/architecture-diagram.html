The Four Layers

Everything in MCAPS IQ lives in one of four layers. Think of it like a building — the top floor handles every request, the lower floors only open when needed.

Tier 0

🏠 Global Rules ✏️ Editable

Loaded on every single request

The "system prompt" — tells Copilot how to behave, what role you are, and how to route your requests. This is the #1 customization lever.

📄 File
📄 .github/copilot-instructions.md
~80 lines · Controls default behavior, role selection, response style
Tier 1

📋 Operational Rules ✏️ Editable

Loaded when your prompt matches keywords in the file's description

Workflow gates, CRM query patterns, entity schemas, write safety rules. These define how operations should be carried out.

📁 Folder
📁 .github/instructions/
~10 files · CRM schema, query strategy, write gates, MCEM flow, shared patterns
Tier 2

🧠 Domain Skills ✏️ Editable

Loaded when your prompt matches the skill's name or description

43 atomic skills — each one handles a specific capability like "milestone health review" or "pipeline hygiene triage." This is where the deep expertise lives.

📁 Folder
📁 .github/skills/*/SKILL.md
43 skills · 30–80 lines each · Role cards, MCEM diagnostics, reviews, sync
Tier 3

🔌 Data Connections ⚙️ Advanced

Always available — these define what data Copilot can access

MCP server definitions, launch scripts, and prompt templates. These connect Copilot to CRM, M365, your vault, and Power BI.

📄 Files
📄 .vscode/mcp.json
📁 .github/prompts/
MCP server registry + 20+ slash-command prompt templates

"I Want to Change…" — Where to Go

Pick the scenario that matches what you want to do. Each card shows you exactly which file to edit.

🎭

Change Copilot's default behavior

How it responds, what role it assumes, what it does first when you chat. This is the starting point for most customizations.
Tier 0 .github/copilot-instructions.md
📝

Add a new workflow rule

E.g., "always check budget before creating a milestone" or "require manager approval for Stage 4 moves." Rules and gates live here.
Tier 1 .github/instructions/your-rule.instructions.md
🧠

Teach Copilot a new capability

Create a new skill — a focused playbook for a specific task like "competitive analysis" or "partner engagement scoring."
Tier 2 .github/skills/your-skill/SKILL.md
🔌

Connect a new data source

Add or modify an MCP server to give Copilot access to a new system — like a custom API, database, or internal tool.
MCP .vscode/mcp.json
⚡

Create a reusable prompt workflow

Build a slash command like /morning-brief or /pipeline-review that runs a multi-step workflow with one command.
Prompts .github/prompts/your-prompt.prompt.md
🏗️

Modify how a role behaves

Adjust the playbook for a specific role — Specialist, SE, CSA, CSAM, AE, ATS, or Sales Director. Each has its own role card.
Tier 2 .github/skills/role-{name}/SKILL.md

How Skill Routing Works

You don't pick skills manually. Copilot reads your prompt and matches keywords against every skill's description to decide which ones to load.

JL
YOUR PROMPT
"Review my pipeline and flag any stale deals before my weekly review"
⚙️
Copilot scans
43 skill descriptions
for keyword matches
Skills Activated
✓
pipeline-hygiene-triage matched: "pipeline", "stale"
✓
risk-surfacing matched: "flag", "risks"
✓
shared-patterns matched: "pipeline", "review"
Skills Skipped
—
processing-presentations no keyword overlap
—
proof-plan-orchestration no keyword overlap
💡 Tip: If a skill isn't activating when you expect it to, check its description field in the SKILL.md frontmatter. Does it contain words you'd naturally use? Add more trigger phrases to improve matching.
